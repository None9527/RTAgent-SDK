# RTAgent Project Memory

## Project Identity

RTAgent is a Go Runtime SDK extracted from ngoagent project experience. Its goal is to provide embeddable runtime capabilities and contracts, not to replicate the full ngoagent product surface.

The target SDK must include a runnable runtime core loop. Public interfaces are the port layer that makes the loop embeddable, testable, and replaceable; they are not the final deliverable by themselves.

## Current Status

- First public Go SDK facade lives in `pkg/rtagent`.
- SDK entry has been shifted toward current ngoagent source architecture: `SubmitRun` for app-level command intake, `Run` for lower-level full-loop execution, `Inspect` for read-side projection, and `Emit` as a journal hook.
- `SubmitRun` now drives the first runnable core loop: run initialization, activity/lease lifecycle, context packet assembly, model provider call, optional tool execution loop, approval suspension, journal events, run completion, and inspect/worldstate projection.
- SDK includes an OpenAI-compatible `ModelProvider` implementation with optional SSE streaming handler and DashScope helper for `qwen3.7-plus`.
- SDK model streaming now has stable public event type constants for text and tool-call deltas; the core loop persists stream deltas as `model.delta` journal events for host consumption.
- SDK core loop now preserves structured model provider failure details through the generic `ModelProviderError` contract in `RuntimeStateProjection.Problem` and `turn.failed` event payloads, including provider, HTTP status, provider code, retry/rate-limit flags, safe-for-model flag, and body preview.
- SDK now includes a first PermissionCenter implementation that gates tool execution, workspace writes, and governance proposals; it supports default, acceptEdits, yolo, allow-once, run/session scoped grants, and run/session scoped all-capability grants.
- PermissionCenter now applies session lifecycle gates before existing-grant or permission-mode shortcuts for non-read-only actions, so stopped/stopping sessions cannot continue SDK side effects through old grants or `acceptEdits`.
- PermissionSnapshot and WorldState capability projection now treat stopped/stopping session grants as inactive for non-read-only actions, matching runtime enforcement while leaving historical grant events in the journal.
- PermissionSnapshot and WorldState governance projection now limit pending approvals to deny-only decisions while a session is `stopping` or `stopped`, so host approval UI does not expose allow choices that runtime lifecycle gates will reject.
- Model/plan approval requests returned by `ModelProvider` are now persisted as `model.approval` continuations and resume through the same `ResolveApproval` path as tool approvals.
- SDK now includes first session lifecycle APIs: `InspectSession`, `SessionGraph`, and `StopSession`; public `SessionID` maps to the existing internal `ThreadRecord.ResumeID`.
- `StopSession(drain)` now rejects new runs while preserving active runs, then automatically marks the session `stopped` once the last active run reaches a terminal status.
- `StopSession(drain)` approval semantics are now contract-covered: non-deny `ResolveApproval`/`ResumeRun` is blocked while the session is `stopping`, records no active grant, and deny remains allowed so a suspended run can terminate and let the drained session reach `stopped`.
- Approval resume is now scope-anchored: `ResumeRun` rejects supplied run/session/root overrides that do not match the original approval continuation before recording grants or executing tools.
- SDK now includes run-internal checkpoint graph/resume: the core loop checkpoints context packet, model request/response, tool call/observation, approval pending, and terminal states; `CheckpointGraph`, `ResumeRun`, and `ResolveApproval` restore loop continuation state.
- `CheckpointGraph.ResumeReady` is now lifecycle-aware: terminal run status or stopped/stopping sessions mark resumable checkpoint nodes unavailable and include warnings, matching `ResumeRun` enforcement.
- `CheckpointGraph` now handles existing runs with no checkpoint records as an empty graph instead of panicking, preserving read-side projection stability.
- SDK config is split into `Config.Runtime` for SDK infrastructure settings and `Config.Host` for host-provided model/tool/memory/hypothesis/MCP/skill/WorldState ports.
- Broad exported `Core` interface has been removed; host integrations should depend on focused public ports or the concrete `Runtime` facade.
- SDK facade now holds an unexported `runtimeKernel` with narrow internal ports instead of concrete startup container dependencies or the aggregate persistence `Bundle`; default startup wiring is confined to kernel bootstrap, and custom kernel/store injection remains an internal seam for v1.
- SDK zero-config `Open(ctx, Config{})` uses isolated in-memory SQLite so embedding the library does not create `rtagent.db` in the host working directory; durable session/resume hosts must pass an explicit `RuntimeConfig.DSN`.
- SDK `Runtime.Close` is idempotent and every public runtime facade method returns the stable sentinel `ErrRuntimeClosed` after close; this lifecycle contract is test-covered.
- `Runtime.Emit` serializes automatic sequence assignment and append inside a single `Runtime` instance; SQLite also enforces unique `(run_id, sequence)` event ordering.
- `Runtime.Emit` now rejects unknown run ids before appending, so host-defined journal events must attach to an existing run/session graph and cannot create orphan runtime facts.
- SDK now includes `ToolRegistry` and `Config.Host.Tools` for multi-provider tool composition while keeping the core loop dependent on a single `ToolProvider` port; duplicate tool short names are exposed through deterministic `namespace__name` aliases.
- Legacy internal PermissionCenter, governed executor, soft sandbox demo wiring, unused internal REST API server, and REST console command have been removed; `cmd/rtagent` is now a public SDK smoke command instead of an internal startup demo.
- SDK now persists tool schema snapshots during context packet assembly and validates supplied tool call schema hash / epoch before execution.
- Tool calls that omit schema metadata are now bound to the current `ToolSpec.SchemaHash` / `ToolSpec.Epoch` before permission checks, execution, and approval continuation persistence.
- SDK WorldState is now a v1 typed, partitioned, source-watermarked read model. It preserves legacy flat `Entries` while adding partitions, handles, capability state, context packet projection, registered context handles, committed memory projection, hypothesis projection, activity/task trajectory, tool-schema-derived capabilities, MCP/Skill inventory projection, governance projection, and grant-aware capability authorization.
- Public SDK examples now use `run() error` plus top-level `log.Fatal` instead of panic-based host integration; local non-persistent examples use temporary runtime directories and clean them up after execution.
- Runtime journal events are the source of truth.
- WorldState is a derived, run-scoped read model, not a truth store.
- `PermissionSnapshot` and `WorldState` now reject unknown run ids instead of fabricating empty projections, keeping read-side state tied to the run/session truth store.
- Run-scoped mutation APIs now reject unknown run ids before writing derived state: `RegisterContextHandle` does not register orphan handles, `CheckPermission` does not create permission requests, grants, or permission events for missing runs, and `ResolvePermission` rejects stale approval records before reviewer decisions or grants are persisted.
- Current implementation is v0.2: it has a deterministic default model provider, separates runtime config from host ports, supports injected model/tool providers and multi-provider tool registries, records permission grants, resumes permission-gated tool approvals, persists tool schema snapshots/checkpoints, exposes session/run/checkpoint graph state, supports OpenAI-compatible streaming/message history, and projects WorldState through dedicated memory/hypothesis/MCP/skill/provider ports.
- SDK now includes lightweight host provider adapters (`ModelProviderFunc`, `ToolProviderAdapter`, memory/hypothesis/MCP/skill provider funcs, and `WorldStateProviderAdapter`) so simple integrations and examples can implement public ports without boilerplate structs.
- SDK host adapter helpers now have explicit nil/empty behavior coverage: function adapters return stable errors instead of panicking, `ToolProviderAdapter` treats nil `Specs` as empty inventory and requires `Execute`, and `WorldStateProviderAdapter` preserves partition/provider metadata when `Build` is missing.
- SDK runtime tests now dogfood public provider adapters (`ModelProviderFunc`, `MemoryProviderFunc`, `HypothesisProviderFunc`, `MCPProviderFunc`, and `SkillProviderFunc`) instead of keeping duplicate local provider func types.
- SDK WorldState implementation has been split by projection responsibility: snapshot assembly, context projection, legacy compatibility projection, shared helpers, memory/hypothesis, capability inventory, external host providers, and runtime governance/activity/task projections now live in separate files instead of one oversized `world_state.go`.
- SDK public schema declarations have been split out of the former monolithic `types.go` into focused files for constants, config, runtime contracts, model, session/checkpoint, permission/plan, tool/workspace, and WorldState contracts while preserving package-level API names.
- SDK PermissionCenter implementation has been split by responsibility: entry methods, local policy normalization, grant/id helpers, persistence/continuations, event emission, and internal record types now live in separate `permission_center_*` files while preserving the public permission API.
- SDK session lifecycle implementation has been split by responsibility: session inspection/record projection, run graph projection, and stop/drain behavior now live in separate `session*.go` files while preserving the public session API.
- SDK Runtime facade implementation has been split by responsibility: lifecycle/config, run intake, journal events, read-side projection, context handles, side-effecting APIs, and shared helpers now live in separate `runtime_*` files while preserving the public `Runtime` API.
- SDK OpenAI-compatible provider implementation has been split by responsibility: public provider/config/HTTP client, internal Chat Completions DTOs, request/message/tool mapping, response/SSE decoding, and provider error mapping now live in separate `openai_provider_*` files while preserving the single `ModelProvider.CompleteTurn` contract.
- SDK core loop implementation has been split by responsibility: run entry/model-turn loop, context packet assembly, tool permission/execution path, run outcome transitions, activity/plan events, and loop-local helpers now live in separate `loop_*` files while preserving checkpoint, permission, and event semantics.
- SDK SQLite persistence adapter implementation has been split by store responsibility: bundle bootstrap, time helpers, run/thread, checkpoint/event, memory/artifact, activity, and permission/governance persistence now live in focused files instead of one oversized `bundle.go`.
- SDK no longer keeps an unused internal message persistence table/store; model history is persisted through checkpoint continuation payloads and runtime events instead of a no-op message store surface.
- SDK no longer keeps an unused internal evidence persistence table/store; evidence remains a public reference/projection concept until a real provider or writer exists.
- SDK no longer keeps unused task persistence or unconsumed Bundle methods such as run/session delete, direct memory get, capability get, and activity list; task WorldState is derived from runtime events.
- SDK no longer migrates or names WorldState as a persistence table; the internal flat compatibility entries now live under the WorldState domain and remain runtime-derived read projections.
- SDK no longer keeps unused internal WorldState query/provider interfaces or unsupported context materializer handle constants; only concrete runtime projection and artifact materialization remain.
- SDK internal WorldState domain no longer duplicates public typed projection partitions; it keeps only the runtime flat-builder partitions used by the legacy compatibility view.
- SDK internal runtime components no longer depend on the aggregate persistence `Bundle`; materializer, workspace, lease manager, and legacy WorldState builder now accept narrow store capabilities matching the methods they actually call.
- SDK facade kernel no longer depends on the aggregate persistence `Bundle`; `runtimeStore` includes only the run/session/checkpoint/event/memory/activity/capability/tool-schema/permission/grant store capabilities used by `pkg/rtagent`, leaving the aggregate `Bundle` in `internal/startup` composition only.
- SDK internal persistence domain contracts have been split by responsibility: source refs, runtime records, memory records, governance records, WorldState records, and store interfaces now live in focused files instead of one monolithic `contracts.go`.
- SDK now has a documented public v1 compatibility policy in `docs/api/public-compatibility.md`; exported stable surfaces, additive-change rules, breaking-change rules, and non-contract internal surfaces are explicit.
- SDK public API surface is now snapshotted with `go doc -all` in `docs/api/public-api.snapshot.txt` and validated by `scripts/check_public_api_snapshot.sh`, including exported fields and method sets.
- SDK public API surface has been narrowed by removing exported facade convenience interfaces for Runtime method groupings; hosts should use concrete `Runtime` or define local narrow interfaces, while SDK-provided host ports remain exported.
- SDK public facade has been narrowed further by removing the misleading `Runtime.Execute` entry; run initialization is internal, `SubmitRun` remains the app-level entry, and `Run` remains the lower-level full-loop entry for hosts that build `RuntimeCommand`.
- SDK tool contract no longer exposes legacy sandbox-demo wording; `ToolSpec.ExecutionConstraints` now carries host-visible execution constraints without implying a bundled SDK sandbox executor.
- SDK provider error boundary has been narrowed: OpenAI-compatible HTTP failures use an internal concrete error that implements the generic `ModelProviderError` contract, so v1 hosts depend on `ModelProviderErrorDetails` rather than provider-specific error structs.
- SDK public schema constants have been narrowed to current v1 outputs; unused legacy v0 RuntimeEvent/WorldState schema constants were removed before the v1 API freeze.
- SDK public request/query structs that commonly cross host process or HTTP/CLI adapter boundaries now use stable snake_case JSON tags, with contract coverage for run intake, runtime command/event drafts, event/inspect/session/checkpoint/worldstate/permission queries, approval resume/stop-session requests, and workspace write requests.
- SDK read-side query semantics now honor session-scoped fields: `ListEvents(EventQuery{SessionID})` aggregates all run events in a session, rejects unknown run/session ids, and `Inspect(InspectQuery{SessionID})` resolves to the latest run while validating mixed RunID/SessionID queries.

## Risks / Known Issues

- Default SDK startup still comes from internal bootstrap; custom kernel/store injection is intentionally not public for v1, and host extension should use `Config.Host` ports.
- Event sequence assignment is serialized inside one `Runtime` instance; multi-process writers sharing the same SQLite DSN are not a committed v1 capability yet.
- The default model provider is a deterministic local placeholder so the SDK loop runs without external credentials; real hosts should inject a `ModelProvider`.
- DashScope real integration uses `DASHSCOPE_API_KEY`; region endpoint can be overridden with `DASHSCOPE_BASE_URL`.
- PermissionCenter policy is v0 and intentionally conservative; host applications still need an approval UI that calls `ResolvePermission` for grant-only resolution or `ResolveApproval` when the suspended SDK run should continue.
- PermissionCenter intentionally does not provide a default unscoped global `allow_all`; process-wide approval bypass remains a host policy or explicit SDK permission mode decision.
- `StopSession` supports `cancel_active` and `drain`; it does not release all possible external tool resources beyond interrupting active SDK runs.
- Generic checkpoint resume is run-internal loop continuation, not product-level multi-run/session replay.
- Zero-config runtime storage is ephemeral by design; examples and persistent hosts should keep passing explicit temp/file DSNs when they need inspect/resume data after process exit.
- ToolRegistry namespaced duplicate routing is provider-order deterministic; hosts should set explicit `ToolSpec.Namespace` when exposed tool names must remain stable across provider ordering changes.
- OpenAI-compatible function calls are bound to the current context packet schema before execution, but ordinary function-call APIs cannot prove which schema version the model actually saw unless the provider returns explicit schema hash/epoch metadata.
- WorldState provider ports are projection-only; hosts must still route execution through ToolProvider and PermissionCenter.
